Draft gameplan rules (reference)

Goal
- Build a 23-player gameplan: 11 starters + 11 substitutes + 1 wildcard.
- Output will mark any empty formation slot as `VACANT`.

Formation (default)
- Slot order (each exactly one player):
  CF, LWF, RWF, AMF, CMF, DMF, LB, CB, CB, RB, GK
- If a different formation is stated in the future, use that instead.

Player selection
- Choose STARTERS first, then choose SUBSTITUTES.
- Starters use this order:
  - Stage A: non-Standard cards (`Epic`/`BigTime`/`Showtime`/`Highlight`) in their `MAIN position`.
  - Stage B: if starter slots are still `VACANT`, non-Standard cards via `proficient_positions`.
  - Stage C: if starter slots are still `VACANT`, Standard cards in their `MAIN position`.
  - `semiproficient_positions` are not used for starters.
- Substitutes use `MAIN position` only during initial selection, except for the `SS` fallback rule for substitute `LWF`/`RWF`.
- If substitute slots remain `VACANT`, fill them in this order:
  - `MAIN` non-Standard
  - `MAIN` Standard
  - `proficient_positions` non-Standard
  - `proficient_positions` Standard
  - `semiproficient_positions` non-Standard
  - `semiproficient_positions` Standard
- The same player_id may not appear more than once anywhere in the gameplan (no duplicates across starters, subs, wildcard).
- Starters are fixed once chosen (do not change starter selection based on substitutes).
- Selection is rating-driven and does not depend on formation order:
  - Within each stage (A/B/C), repeatedly assign the single highest-rated eligible role across all currently VACANT slots.
  - A new stage begins only when no further assignments are possible in the previous stage.

- Substitute-specific SS fallback:
  - Only for SUBSTITUTE `LWF` or `RWF` vacancies: you may use an `SS` player, but only when no eligible non-Standard direct roles exist for that slot.
  - Do not use `SS` to fill substitutes in the Standard stage.
  - More generally, substitute slots that cannot be satisfied directly by `MAIN position` (for example `SS`) are lower priority than direct `MAIN-position` substitute assignments.

Jersey number assignment (overall)
- Assign jerseys after players are selected.
- If a player cannot be assigned ANY available jersey number, that player is eliminated from the gameplan.
- Starters have higher priority than substitutes in jersey assignment.
  - Assign starter jerseys first.
  - Then assign substitute jerseys from the remaining numbers.
  - A substitute with recent=True cannot override a starter’s jersey.
- Standard players do not override already-assigned numbers, even if recent=True; they only compete within the Standard stage.

Jersey number assignment priority & preferences
1) recent=True lock (within the group being assigned)
   - If recent is marked True, that player secures the MOST RECENT jersey number they have worn (newest → oldest).

2) Card-type priority (after applying recent=True locks)
   - Epic
   - BigTime
   - Showtime
   - Highlight
   - Standard

Additional ordering notes
- SS used as SUBSTITUTE LWF/RWF (vacancy fill) has lower priority than already-assigned Epic/Showtime/Highlight players,
  but higher priority than Standard players.
- Wildcard has the lowest priority overall (after substitutes, including substitute Standards).
- Wildcard may be any remaining player; it is chosen only after starters + substitutes are finalized.

3) How each card-type chooses numbers
  - BigTime:
    - Treated as Epic-tier for jersey-assignment priority.
    - Preference list is the player's jersey history order (newest → oldest), with duplicates removed while preserving order.
  - Epic:
    - Preference list is based on MOST-WORN jersey number over all years.
    - If two jersey numbers have the same total wear count, prefer the one that appeared earlier in jersey history (newest first).
    - If there is a conflict, higher rating keeps the number and the lower-rated player moves to their next preference.
    - A player's "next-most-worn" does not outrank another player's "most-worn".
  - Showtime / Highlight / Standard:
    - Follow the same conflict-resolution logic as Epic, but preference is based on the MOST FREQUENT number worn RECENTLY.

Wildcard rule
- After all starter + substitute jersey numbers are assigned, select the wildcard:
  - Pick the highest-rated player not already in the gameplan (still respecting unique player_id).
- Then assign the wildcard a jersey number:
  - Assign using the same jersey preference rules as their card type (must be a number they actually wore).
  - If none of their worn numbers are available, try the next-highest-rated remaining player, and so on, until one works.

Starter replacement rule (when jerseys fail)
- If a selected starter cannot be assigned ANY available jersey number:
  - First try promoting the best already-selected substitute whose MAIN position matches that starter slot, and backfill that cleared substitute slot later.
  - If promotion is not possible, replace the starter with the best next available candidate for that slot.

- Vacant slot fill
  - After starter + substitute selection/jersey assignment, if any formation slot is still empty, try to fill it from players that were not selected in starters/subs and do not have a jersey assigned yet.
  - First try `proficient_positions`.
  - Then try Standard players in `MAIN position`.
  - Jersey priority for proficient-position fills is lower than all earlier card-type jersey assignment, but higher than Standard fills.
  - If still no candidate works, the slot remains `VACANT` in the output.

Data source
- All necessary information is pulled from the SQLite database (players, game_data, jersey history).
