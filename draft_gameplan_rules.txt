Draft gameplan rules (reference)

Goal
- Build a 23-player gameplan: 11 starters + 11 substitutes + 1 wildcard.
- Output will mark any empty formation slot as `VACANT`.

Formation (default)
- Slot order (each exactly one player):
  CF, LWF, RWF, AMF, CMF, DMF, LB, CB, CB, RB, GK
- Override: `{country}_formation.txt` in the country folder (comma-separated positions); if missing, the default above is used.

Player selection
- Choose STARTERS first, then choose SUBSTITUTES.
- Starters use this order:
  - Stage A: non-Standard cards (`Epic`/`BigTime`/`Showtime`/`Highlight`) in their `MAIN position`.
  - Stage B: if starter slots are still `VACANT`, non-Standard cards via `proficient_positions`.
  - Stage C: if starter slots are still `VACANT`, Standard cards in their `MAIN position`.
  - Stage D: if starter slots are still `VACANT`, Standard cards via `proficient_positions`.
  - `semiproficient_positions` are not used for starters.
- Initial substitutes: same rating-driven “pick the best eligible player for any open sub slot” pattern as starters, but each slot only considers `MAIN position` matches; for sub `LWF`/`RWF`, if there is no non-Standard direct winger, non-Standard `SS` may be used (Standard wings and Standard `SS` are not used in that fallback).
- If substitute slots are still `VACANT` after lineup + jersey processing, fill them in this order:
  - `MAIN` non-Standard
  - `MAIN` Standard
  - `proficient_positions` non-Standard
  - `proficient_positions` Standard
  - `semiproficient_positions` non-Standard
  - `semiproficient_positions` Standard
- The same player_id may not appear more than once anywhere in the gameplan (no duplicates across starters, subs, wildcard).
- Initial starters are chosen before the first substitute pass; after that, starters may still change (jersey failures, promotions, vacancy fill) while honoring uniqueness.
- Selection is rating-driven and does not depend on formation order:
  - Within each starter stage (A/B/C/D), repeatedly assign the single highest-rated eligible role across all currently VACANT starter slots.
  - A new stage begins only when no further assignments are possible in the previous stage.

- Substitute-specific SS fallback:
  - Only for SUBSTITUTE `LWF` or `RWF` vacancies: you may use an `SS` player, but only when no eligible non-Standard direct roles exist for that slot.
  - Do not use `SS` to fill substitutes in the Standard stage.
  - More generally, substitute slots that cannot be satisfied directly by `MAIN position` (for example `SS`) are lower priority than direct `MAIN-position` substitute assignments.

Jersey number assignment (overall)
- Assign jerseys after players are selected.
- If a player cannot be assigned ANY available jersey number, that player is eliminated from the gameplan.
- Jersey assignment order by group (each group runs the same internal recent-lock + card-tier logic):
  - non-Standard starters, then non-Standard substitutes (excluding SS-on-wing subs), then non-Standard substitutes slotted as `SS` on `LWF`/`RWF`, then Standard starters, then Standard substitutes.
  - So a Standard starter is assigned after those non-Standard substitute groups (numbers already taken stay taken).
- A substitute with recent=True cannot take a number already assigned to someone else; it does not override starters.

Jersey number assignment priority & preferences
1) recent=True lock (within the group being assigned)
   - If recent is marked True, that player secures the MOST RECENT jersey number they have worn (newest → oldest).

2) Card-type priority (after applying recent=True locks)
   - Epic and BigTime share the highest tier in code (then Showtime, then Highlight, then Standard).

Additional ordering notes
- Within the non-Standard substitute jersey phase, a sub playing `SS` on a wing slot is assigned after other non-Standard subs but before any Standard starter/sub jersey assignment.
- Wildcard has the lowest priority overall (after starters + subs are resolved for the main 22).
- Wildcard may be any remaining player; it is chosen only after starters + substitutes are finalized.

3) How each card-type chooses numbers
  - Epic:
    - Preference list is based on MOST-WORN jersey number over all stored history.
    - Tie-break among equal wear counts: earlier in stored jersey order (newest entries first in the DB ordering used by the script).
  - BigTime / Showtime / Highlight / Standard:
    - **Number preference list** (all except Epic): jersey history order from the DB (newest → older), with duplicates removed while preserving that order (BigTime, Showtime, Highlight, and Standard use this same preference shape in code).
  - Within a jersey group, a lower-priority player will not take a number that is still another (same-group) player’s first choice when that tie-break applies.

Wildcard rule
- After all starter + substitute jersey numbers are assigned, select the wildcard:
  - Pick the highest-rated player not already in the gameplan (still respecting unique player_id).
- Then assign the wildcard a jersey number:
  - Assign using the same jersey preference rules as their card type (must be a number they actually wore).
  - If none of their worn numbers are available, try the next-highest-rated remaining player, and so on, until one works.

Starter replacement rule (when jerseys fail)
- If a selected starter cannot be assigned ANY available jersey number:
  - First try promoting the best already-selected substitute whose MAIN position matches that starter slot, and backfill that cleared substitute slot later.
  - If promotion is not possible, replace the starter with the best next available candidate for that slot.

- Vacant slot fill (starters)
  - After starter + substitute selection/jersey assignment, empty starter slots: non-Standard via `proficient_positions`, then Standard via `proficient_positions`, then Standard in `MAIN position` (can pull from bench roles where allowed by code). Numbers already on starters block picking duplicates for those fills.
  - If still no candidate works, the slot stays empty (`VACANT` in the output).
- Vacant sub slots may also use an extra swap routine in code (reassign another sub to free a number) before remaining `VACANT`.

Data source
- All necessary information is pulled from the SQLite database (players, game_data, jersey history).
